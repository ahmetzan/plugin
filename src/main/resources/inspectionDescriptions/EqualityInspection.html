This inspection reports usages of == and != operators when comparing wrapper classes (e.g. Integer, String) or Java collections (List, Set, Map).

Using == or != on such types may lead to incorrect behavior because it checks reference equality rather than value or emptiness.

Reported cases

  • Wrapper types:
    • a == null → suggest Objects.isNull(a)
    • a != null → suggest Objects.nonNull(a)
    • a == x or a != x → suggest Objects.equals(a, x)
  • Collection types:
    • list == null → suggest CollectionUtils.isEmpty(list)
    • list != null → suggest CollectionUtils.isNotEmpty(list)

Quick Fix

A quick fix is available to automatically replace the suspicious comparison with the appropriate utility method from java.util.Objects or org.apache.commons.collections4.CollectionUtils.
